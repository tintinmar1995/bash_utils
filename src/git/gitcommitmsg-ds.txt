### HELP FOR PREFIXES ###
# feat : Create a feature.
# test : Create (unit) tests.
# config : Adjust dependencies or some config files.
# model : Adjust hyperparameters.
# drop : Delete some feature or test.
# fix : Fix an issue e.g. bug, typo, accident, misstatement.
# optim : A change that MUST be just about performance, e.g. speed up code.
# doc : A change that MUST be only in the documentation, e.g. help files.
# wording : A change that MUST be just refactoring/formating/rephrasing, e.g. indent line, trim space, edit a comment, doc, etc.
